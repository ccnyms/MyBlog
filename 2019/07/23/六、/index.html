预编译

JS 运行执行步骤

  1. 语法分析（通篇扫描一遍，看看有没有啥错误）
  2. 预编译
  3. 解释执行

全局变量

  1. 暗示全局变量（imply global）：任何变量，如果未声明就赋值，那么此变量就为全局对象（window）所有。

  2. 所有声明了的全局变量，也为 window 所有。

  3. 上面两种都是全局对象 window 的属性（属性名+属性值），window 相当于是全局的域。

预编译过程

函数内

  1. 函数内的预编译发生在函数执行的前一刻。

  2. 步骤：找形参和变量声明，值为 undefined -> 实参值传给形参 -> 找函数声明。

    • 创建 AO 对象（Activation Object）：执行期上下文，可理解为作用域。

    • 找形式参数和变量声明，将形式参数名和变量名作为 AO 属性名，值为 undefined。（变量声明提升）

    • 将实际参数和形式参数统一。
    • 在函数体里找函数声明，将函数名作为 AO 属性名，值为自身的函数体（函数声明提升）
    • 执行。

全局内

  1. 全局内的预编译发生在全局执行的前一刻。
  2. 步骤：找变量声明，值为 undefined -> 找函数声明。
    • 创建 GO 对象（Global Object）：全局执行期上下文，可理解为作用域。（GO 就是 window ）
    • 找变量声明，将变量名作为 GO 属性名，值为 undefined。（变量声明提升）。
    • 找函数声明，将函数名作为 GO 属性名，值为自身的函数体（函数声明提升）。
    • 执行。
  3. GO 发生在全局执行前，AO 发生在函数执行前，如果 AO 内找不到，会直接在 GO 中找。

补充

  1. 通过函数声明定义的函数，会提升到脚本块的顶部。
  2. 通过函数声明定义的函数，会成为全局对象的属性。
  3. 通过函数表达式定义的函数，它既不会提升，也不会污染全局对象。
